# Editing project information

All public project records are stored in:

`data/projects.json`

## Edit on GitHub

1. Open `data/projects.json`.
2. Select the pencil icon.
3. Search for the stable project ID, such as `GVG-PAVE-DES-001`.
4. Edit the public fields.
5. Preserve quotation marks, commas, square brackets and braces.
6. Commit the change. GitHub Pages will redeploy automatically.

## Public fields

- `summary` — What we are building
- `currentMilestone` — Progress and current activity
- `nextDecision` — Next measurable outcome
- `keyRisk` — What must be in place
- `supporterAsk` — Ways to participate
- `supportOutcome` — What support will unlock
- `publicBadges` — Up to three short proof/opportunity badges
- `publicUrl` — Official public website

## Rules

- Never change or reuse a stable `id`.
- Do not publish internal cash, private Drive links, cap tables, personal data,
  draft token terms or unapproved investment claims.
- Describe future results as targets or milestones, not guarantees.
- Keep asks specific and actionable.
- Do not add trailing commas.
